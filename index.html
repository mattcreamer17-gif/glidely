The question that gets harder as the customer count grows is context.

Glidely is pulling contracts from Ironclad, spend from Brex, tasks from Jira, and stakeholder input from Slack into one agent workflow. The renewal agent flags notice windows, surfaces seat waste, and drops the next action back into Teams. The new-purchasing agent captures requirements, generates RFXs, and starts supplier outreach. Ten POCs so far, 40% shorter cycles. Seven figures in vendor spend moving through the product each month.

The question that gets harder as the customer count grows is context. Not prompts. Context. Which contract did the agent read, in what order, under which interpretation of the stakeholder's intake interview. When a customer calls six weeks in asking why the agent surfaced vendor X and not vendor Y, the answer has to be reproducible and auditable without the founders in the loop.

Eduba thinks about AI systems in three layers. About 60% of the work is traditional code and databases, the kind of plumbing that keeps integrations alive. Another 30% is rule-based logic, the conditionals that decide which agent wakes up when and what it is allowed to touch. The remaining 10% is genuine AI work, where language models do something a rule cannot. Teams that do not draw those lines end up putting LLMs where Postgres would do the job faster, cheaper, and with better guarantees. For Glidely, the 10% is real, and the context-management layer that makes the 10% debuggable is where most of the engineering leverage sits.

Feeld (the product company, Andrew Santus, CTO) ran a scoped sprint with us on exactly this problem. Workshop, advisory calls, an Organizational Context Architecture their team still uses. Deliverable was not a slide deck. It was a folder structure and a set of rules for how their agents read and write context, so the next engineer they hired could understand the system without a month of ramp.

The underlying method is published: Interpretable Context Methodology, submitted to ACM TiiS, MIT-licensed on GitHub, with a 52-member practitioner community. Written for exactly the kind of agent system Glidely is building.

Jake also built an online community to 22,000 members in five weeks. Separate track, useful context on distribution instinct.
